KEY PROPERTIES
==============

1. TARGET DISCOVERY IS NON-DISRUPTIVE
   • It's a PRE-PROCESSOR that runs before load_leads()
   • By the time your pipeline touches the data, __target__ exists
   • Zero changes to any downstream function

2. CLIENT NEVER SEES COMPLEXITY
   • Upload CSV
   • See 3-5 clear ranking options
   • Pick one → model trains
   • No manual feature selection, column mapping, or config

3. FULLY UNIVERSAL
   • Works on CSV with 5 columns or 50
   • Works on any schema (Salesforce, HubSpot, custom)
   • Same code, any customer, any data structure


EXAMPLE: THREE DIFFERENT CUSTOMERS

Customer 1 (Salesforce):
   has: Account__c, Amount, Stage__c, LeadScore
   discovery: Finds Stage__c binary target → Proceeds normally

Customer 2 (HubSpot):
   has: Deal Value, Lead Status, Days in Cycle, Company Size
   discovery: No binary target → Shows options → Client picks "High Deal Value"

Customer 3 (Custom):
   has: revenue, status_code, last_activity, company
   discovery: No binary target → Shows options → Client picks "Most Recent"

THE SAME CODE HANDLES ALL THREE. No branching. No special cases.
